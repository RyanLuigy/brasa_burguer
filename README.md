# 🍔 Brasa Burguer - Hamburgueria Artesanal

Mini-site empreendedor simples, moderno e responsivo desenvolvido como projeto prático para a disciplina de Desenvolvimento Full-Stack. O projeto simula a presença online de uma hamburgueria focada no sabor rústico da brasa.

## 🚀 Demonstração

O site está publicado e pode ser acessado através do link abaixo:
**[CLIQUE AQUI PARA ACESSAR O SITE NO VERCEL](https://brasaburguer.vercel.app/)**

---

## 📋 Requisitos do Projeto

O desenvolvimento foi dividido em três etapas fundamentais:
1. **Construção das Páginas (HTML):** Estruturação semântica de 3 páginas (Home, Sobre e Contato).
2. **Estilização (CSS):** Layout centralizado, uso de Flexbox/Grid, efeitos hover, sombras e design responsivo.
3. **Versionamento e Deploy:** Uso de Git para controle de versão, GitHub para hospedagem do código e Vercel para publicação.

---

## 💻 Tecnologias Utilizadas

* **HTML5**: Estruturação semântica.
* **CSS3**: Estilização, animações hover e responsividade.

---

## 📂 Estrutura de Páginas

### 1. Home (`index.html`)
* **Hero Section**: Título de impacto e Chamada para Ação (CTA).
* **Cards de Benefícios**: Seção destacando diferenciais como Ingredientes Premium e Entrega Rápida.
* **Layout**: Design focado na conversão do cliente.

### 2. Sobre (`sobre.html`)
* **História**: Breve relato sobre a origem da Brasa Burguer.
* **MVV**: Missão, Visão e Valores apresentados em cards estilizados.

### 3. Contato (`contato.html`)
* **Formulário**: Campos para Nome, E-mail e Mensagem.
* **Links Diretos**: Integração com `mailto`, `tel` e link externo para Redes Sociais.

---

## 📱 Responsividade

O site foi desenvolvido seguindo o conceito de *Mobile First* (ou adaptabilidade), garantindo que os elementos e cards sejam empilhados em telas menores para uma leitura fluida em dispositivos móveis.

---

## 🛠 Histórico de Commits

Para manter a organização do desenvolvimento, foram realizados os seguintes commits principais:
1. `Estrutura inicial HTML`: Criação dos arquivos base e conteúdo semântico.
2. `Menu e rodapé`: Padronização da navegação global e footer.
3. `CSS base`: Implementação de cores, tipografia e layout dos cards.
4. `Responsividade e ajustes finais`: Media queries e refinamento de design.
